  Design Proposal: Port Configuration via CLI Flags

  Overview

  Add -p / --port flags to the existing devbox up command to allow users to
  expose different or multiple ports when starting their devbox container.

  Approach

  Recommended: Extend the existing devbox up command with port flags using
  clap's argument parsing.

  Why this approach:
  - Minimal changes to CLI structure - users don't need to learn new commands
  - Consistent with Docker conventions (docker run -p ...)
  - Supports multiple ports via repeated flags
  - Simple and intuitive syntax

  Design Details

  CLI Changes (main.rs / commands.rs)

  Add optional port arguments to the Up command:
  #[derive(Subcommand)]
  enum Commands {
      /// Create or attach to a devbox environment
      Up {
          #[arg(short = 'p', long = "port", value_name = "HOST:CONTAINER")]
          ports: Vec<String>,
      },
  }

  Usage Examples

  # Single port mapping
  devbox up -p 8080:3000

  # Multiple ports
  devbox up -p 8080:3000 -p 5000:5000

  # Mix of ports (web + API)
  devbox up -p 80:3000 -p 9229:9229 -p 5432:5432

  Container Creation (backend.rs)

  Modify create_container to accept a vector of port mappings and iterate
  through them, adding -p flags for each mapping when running the docker
  container.

  ---
  Does this design look right? I can proceed with creating a design document and
   implementation plan once you confirm.

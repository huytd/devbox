❯ will using tail block me from connect to the container to work on it? 

● No, using tail -f /dev/null won't block you from connecting to the container.
  You can still attach to it using docker exec -it <container_name> bash, which
  is exactly what the file:///home/king/code/devbox/src/backend.rs:81 method
  does.

  The flow would be:
  1. Container starts with tail -f /dev/null (keeps it running in background)
  2. You run devbox up which calls attach_container
  3. That runs docker exec -it <container_name> bash to give you an interactive
  shell

  So you'd still get full access to the container's bash shell for development
  work. The container stays running in the background, and when you exit the
  shell (by typing exit), it just closes your session but keeps the container
  alive.
